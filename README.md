\# DAD 220 SQL Analysis Project



This project recreates a relational database and analysis workflow originally completed for DAD 220. The database includes customer, order, and return-related data and demonstrates table creation, CSV imports, and SQL analysis queries.



\## Project Overview



The project uses three tables:



\- `Customers`

\- `Orders`

\- `RMA`



The `Customers` and `Orders` tables were rebuilt from original coursework CSV files. The `RMA` table was recreated using a small sample dataset because the original `rma.csv` file from Codio was not retained.



\## Skills Demonstrated



\- Creating relational database tables

\- Defining primary keys and foreign keys

\- Importing CSV data into database tables

\- Joining related tables

\- Grouping and aggregating data

\- Ordering query results

\- Calculating return percentages



\## Files Included



\- `database\_setup.sql` — creates the Customers, Orders, and RMA tables

\- `analysis\_queries.sql` — contains the main SQL analysis queries

\- `sample\_rma\_data.sql` — inserts a small sample return dataset into the RMA table

\- `customers.csv` — customer data used for import

\- `orders.csv` — order data used for import



\## Example Analysis Questions



1\. Which states had the highest number of returns?

2\. Which product descriptions had the highest return percentages?



\## Notes



This project is a reconstructed version of coursework originally completed in Codio. Because the original `rma.csv` file was unavailable, a sample RMA dataset was created for demonstration purposes. The project is intended to show SQL skills and relational database understanding in a clean, reproducible format.
